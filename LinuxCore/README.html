  • lckc-find-line.el An Emacs Lisp file that defines a simple Emacs Lisp function, lckc-find-line, which can make it easier to find a line of code in lckc_code given the line number. To use it:
  1. Copy the contents of lckc-find-line.el into your .emacs file.
  2. In a new Emacs (one that has read the updated .emacs file), open lckc_code, then type the following: 
    Esc x lckc-find-line

    and type Return.

  3. Emacs prompts you for the line number. Type in the line number, followed by Return. Emacs jumps to the given line. (Note to Emacs aficionados: This is not the same as goto-line, because there are lines in the file that don't have line numbers.)

Tags Files

Except for the version 0.01 kernel, I've added tags files to all of the kernel distributions. Various editors can use tags files to jump directly to a function or variable definition, to perform intelligent searches through a set of source files, and so on.

Inside each of the 2.x distribution directories, you'll find a file named TAGS.all, which was built by running the etags program over every source file in the distribution. There are also files named TAGS.alpha, TAGS.arm, TAGS.i386, and so on; each of these was built by running etags on the code specific to a single port of the kernel and all of the architecture-independent code. For instance, if you're interested in searching both the architecture-independent code and the i386-specific (that is, Intel-specific) code, use TAGS.i386.

The files that start with "TAGS" are Emacs-specific. The vi-specific versions of the same files have the same names, except that the files are named "vitags" instead of "TAGS" (e.g., use"vitags.i386" for vi, "TAGS.i386" for Emacs).

Using Tags Files In Emacs

In Emacs (my editor of choice), you can jump to the definition of the PAGE_SIZE macro with the following procedure:

  1. Type 
    Esc .

    (that is, type the Escape key followed by a period).

  2. Emacs prompts for the tag to locate. Type PAGE_SIZE and hit Return.
  3. Emacs prompts for the location of the tags file. Type in the path to the tags file and hit Return.

That's it--Emacs opens the correct source file and places the cursor on the line where PAGE_SIZE is defined. After that, if you search for another tag, it's the same procedure, except that Emacs doesn't prompt you for the tags file again. It assumes you want to keep using the tags file until you tell it otherwise with the visit-tags-table command.

Emacs can also search for all occurrences of a tag:

  1. Type 
    Esc x tags-search

    followed by Return.

  2. Emacs prompts for the tag to seek. Type in the tag, followed by Return.
  3. Emacs locates the first occurrence of the tag (not necessarily its definition), opens the source file, and places the cursor just after the tag.
  4. To search for the next occurrence of the tag, type 
    Esc ,

    (that is, the Escape key followed by a comma). You can do this repeatedly; Emacs will beep when there are no more occurrences of the tag.

The tags-search command actually searches all text in the files that were used to build the tags file, even text in comments. What's more, it searches for a regular expression, not just literal text. Consult Emacs's online documentation for more information about Emacs regular expressions.

Using Tags Files In Vi

  1. Tell vi to load the tags file: 
    :set tags=tagfilename

    Of course, you should replace tagfilename with the name of the tags file you want to use. Don't forget to include the path to the file, if necessary (that is, if the tags file isn't in the directory from which you started vi).

    To find the PAGE_SIZE macro, type the following: 

    :ta PAGE_SIZE

As you'd expect, once the tags file is loaded, you don't need to repeat Step 1 every time.

Both Emacs and vi have more sophisticated tags-related functions, which you can learn about by reading the editors' documentation.
